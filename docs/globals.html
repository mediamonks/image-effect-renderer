generateEventTypes

  • generateEventTypes(targets: object): void
    • Defined in lib/util/eventTypeUtils.ts:74

    It is common practice to have static properties on an event class that indicate which event types should be used with that class. For example:

    // FooEvent.ts
class FooEvent extends AbstractEvent {
   ...
   public static COMPLETE:string = 'complete';
   ...
}
// BarEvent.ts
class BarEvent extends AbstractEvent {
   ...
   public static COMPLETE:string = 'complete';
   ...
}

    One problem with this example is that all event handlers added using EventDispatcher.addEventListener for FooEvent.COMPLETE will also be called when the unrelated BarEvent.COMPLETE is dispatched. To prevent this it is best practice to prefix the event type with the event class, like so:

    // FooEvent.ts
class FooEvent extends AbstractEvent {
   ...
   public static COMPLETE:string = 'FooEvent/COMPLETE';
   ...
}
// BarEvent.ts
class BarEvent extends AbstractEvent {
   ...
   public static COMPLETE:string = 'BarEvent/COMPLETE';
   ...
}

    This utility provides a way to generate these event types, making sure that the event types stay consistent throughout your application and that no string is mistyped by accident. Using this utility, our example now looks like this:

    // FooEvent.ts
import {generateEventTypes, EVENT_TYPE_PLACEHOLDER} from 'seng-event/lib/util/eventTypeUtils';
class FooEvent extends AbstractEvent {
   ...
   public static COMPLETE:string = EVENT_TYPE_PLACEHOLDER;
   ...
}
generateEventTypes({FooEvent});
// BarEvent.ts
import {generateEventTypes, EVENT_TYPE_PLACEHOLDER} from 'seng-event/lib/util/eventTypeUtils';
class BarEvent extends AbstractEvent {
   ...
   public static COMPLETE:string = EVENT_TYPE_PLACEHOLDER;
   ...
}
generateEventTypes({BarEvent});

    What this util technically does is it loops through the given object's properties and replaces values with ClassName/propertyName every time it encounters a value equal to EVENT_TYPE_PLACEHOLDER.

    The event class CommonEvent that is included with seng-event also uses this utility to generate event names.

    Parameters

    • targets: object

      An object containing the classes on which types should be generated. Because we cannot reliably detect the name of any given Class, you have to pass an object with the names of the Classes as keys and the classes themselves as values: { 'Class1' : Class1, 'Class2' : Class2 }. Using the new ES6 property names shorthand, we can simplify that to the following: { Class1, Class2 }.

      • [name: string]: Object

    Returns void

removeListenersFrom

  • removeListenersFrom(listeners: EventListenerMap, eventType?: string, handler?: EventHandler, useCapture?: boolean): void
    • Defined in lib/EventDispatcher.ts:275

    Helper function for EventDispatcher.removeEventListener and EventDispatcher.removeAllEventListeners. Will remove all event listeners that match the given parameters from the given event listener map object. This function differs from removeEventListener() in that it does not use default values when you emit one of the parameters. Instead, it will remove event listeners of all possible values for that parameter.

    Parameters

    • listeners: EventListenerMap

      A map of listeners to remove from. See EventDispatcher.listeners

    • Optional eventType: string

      If set, will only remove listeners added with this eventType

    • Optional handler: EventHandler

      If set, will only remove listeners with this handler

    • Optional useCapture: boolean

      If set, will only remove listeners with the same value for useCapture

    Returns void
